Laravel Sanctum: API Authentication Made Simple

Feb 16, 2024 Jayesh Sukhadiya

Introduction

Laravel Sanctum provides a lightweight authentication system for SPAs (Single Page Applications), mobile applications, and simple token-based APIs. This guide will show you how to implement and use Sanctum effectively in your Laravel applications.

Prerequisites

  • Laravel 8.x or higher
  • Composer installed
  • Basic understanding of Laravel authentication

Step 1: Installation

Install Laravel Sanctum via Composer:

composer require laravel/sanctum

Step 2: Configuration

Publish the Sanctum configuration and migration files:

php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate

Step 3: User Model Setup

Add the HasApiTokens trait to your User model:

use Laravel\Sanctum\HasApiTokens;

class User extends Authenticatable
{
    use HasApiTokens, HasFactory, Notifiable;
    
    // Your model code here
}

Step 4: Token Creation

Create tokens for your users:

// Create a token
$token = $user->createToken('token-name');

// Create a token with abilities
$token = $user->createToken('token-name', ['server:update']);

// Access the plain text token
$plainTextToken = $token->plainTextToken;

Step 5: Protecting Routes

Protect your API routes using Sanctum middleware:

// In routes/api.php
Route::middleware('auth:sanctum')->get('/user', function (Request $request) {
    return $request->user();
});

// Protecting specific routes with abilities
Route::middleware('auth:sanctum')->get('/server', function (Request $request) {
    // Only users with the server:update ability can access this route
})->middleware('ability:server:update');

Best Practices and Tips

  • Always use HTTPS in production
  • Implement token expiration
  • Use specific abilities for fine-grained access control
  • Regularly clean up expired tokens

Advanced Features

// Token Abilities
$token = $user->createToken('token-name', ['server:update', 'server:delete']);

// Checking Token Abilities
if ($user->tokenCan('server:update')) {
    // User has the server:update ability
}

// Revoking Tokens
$user->tokens()->delete(); // Delete all tokens
$user->currentAccessToken()->delete(); // Delete current token

Conclusion

Laravel Sanctum provides a powerful yet simple way to handle API authentication in your Laravel applications. By following this guide, you can implement secure token-based authentication for your SPAs, mobile apps, and APIs.
